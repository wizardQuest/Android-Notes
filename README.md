## üìñ Study Strategy

### 1. **Active Reading Approach**

- ‚úÖ **Don't just read** - Write code examples as you go
- ‚úÖ **Take notes** - Summarize key concepts in your own words
- ‚úÖ **Practice immediately** - After each section, try to explain it to someone (or yourself)
- ‚úÖ **Code along** - Create small projects to test concepts

### 2. **Priority Order (If Time is Limited)**

1. **Must Know** (Core for any Android interview):
   - Android.md (Sections 1-3, 13)
   - coroutines/Coroutines.md
   - dependency-injection/Dagger.md (Hilt section)

2. **Should Know** (Important for senior roles):
   - Kotlin.md (Advanced sections)
   - Android.md (System Design section)
   - Gradle.md

3. **Nice to Know** (For lead/architect roles):
   - Advanced Kotlin topics
   - Performance deep dives
   - Build system optimization

### 3. **Practice Strategy**

- **Day 1-7**: Focus on reading and understanding
- **Day 8-10**: Practice coding examples
- **Day 11-12**: Solve tricky questions
- **Day 13-14**: Mock interviews and final review

### 4. **Interview Preparation Checklist**

Before your interview, ensure you can:

- [ ] Explain Activity/Fragment lifecycle without notes
- [ ] Describe MVVM vs MVI vs Clean Architecture
- [ ] Write coroutine code with proper error handling
- [ ] Explain Hilt/Dagger setup and scopes
- [ ] Discuss system design for a mobile app
- [ ] Explain Compose recomposition and stability
- [ ] Describe memory leak scenarios and solutions
- [ ] Explain Gradle build variants and optimization

## üéì Topic-Specific Study Guides

### Coroutines

**Reading Order:**
1. Start with `coroutines/Coroutines.md` - Read sequentially
2. Practice with `TrickyInterviewQuestions.md` (Coroutines section)
3. Review advanced patterns in `Android.md` (Section 5)

**Key Focus Areas:**
- Scopes (viewModelScope, lifecycleScope, GlobalScope)
- Dispatchers (Main, IO, Default)
- Exception handling and cancellation
- Testing coroutines

**Time: 3-4 hours reading + 1-2 hours practice**

---

### Dependency Injection

**Reading Order:**
1. Read `dependency-injection/Dagger.md` from start to finish
2. Focus on Hilt section (most relevant for modern Android)
3. Review multi-module setup patterns

**Key Focus Areas:**
- Hilt setup and components
- Scopes and lifecycle
- Multi-module DI architecture
- Testing with DI

**Time: 3-4 hours reading + 1 hour practice**

---

### Jetpack Compose

**Reading Order:**
1. Read `Android.md` Section 13 (Jetpack Compose)
2. Practice with `TrickyInterviewQuestions.md` (Compose section)
3. Review stability and performance sections

**Key Focus Areas:**
- State management (State, MutableState)
- Recomposition and stability
- Side effects (LaunchedEffect, DisposableEffect)
- Navigation in Compose

**Time: 3-4 hours reading + 2-3 hours practice**

---

### System Design

**Reading Order:**
1. Read `Android.md` Section 17 (Android System Design)
2. Study all 5 complete app designs
3. Practice explaining architecture decisions

**Key Focus Areas:**
- Architecture patterns selection
- Scalability considerations
- Data flow and state management
- Trade-offs and decision-making

**Time: 3-4 hours reading + 2-3 hours practice**

## üí° Tips for Effective Learning

### 1. **Use Spaced Repetition**
- Review topics after 1 day, 3 days, and 1 week
- Focus more time on topics you find challenging

### 2. **Practice Explaining**
- After reading a section, explain it out loud
- Write a summary in your own words
- Teach the concept to someone else (or record yourself)

### 3. **Code Examples**
- Don't just read code - type it out
- Modify examples to test your understanding
- Create your own examples

### 4. **Focus on Understanding "Why"**
- Don't memorize - understand the reasoning
- Know when to use each pattern/tool
- Understand trade-offs

### 5. **Mock Interviews**
- Practice explaining concepts under time pressure
- Get feedback on your explanations
- Practice system design discussions
